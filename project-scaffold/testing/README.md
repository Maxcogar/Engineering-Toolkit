# Testing Directory

Test data, results, and analysis.

## Structure

```
testing/
├── data/       # Raw test data
├── results/    # Processed results
└── analysis/   # Analysis scripts and reports
```

## Workflow

1. Plan test in `analysis/test-plan.md`
2. Collect raw data to `data/`
3. Process and analyze
4. Document results in `results/`

## File Naming

- Raw data: `YYYY-MM-DD_test-name_raw.csv`
- Results: `YYYY-MM-DD_test-name_results.md`
- Analysis: `test-name_analysis.ipynb`

## Documentation

Use `templates/test-results-template.md` for consistent reporting.
